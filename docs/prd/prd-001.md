# Product Requirements Document (PRD)

## 1. Meta

- **Project Name:** Pet Routine
- **Status:** Draft
- **Last Updated:** 2026-06-03
- **Id:** PRD-001

## 2. Problem Statement & Value Proposition

- **Problem:** I have 3 pets at home and I want to create a system that help me to take care of them, remind me of feeding times, walks, and also want to be able to track their health and well-being.
- **Value:**
  - Provides reminders for feeding times, walks, and other pet care activities.
  - Allows tracking of pet health and well-being.

## 3. Target Audience

- **Persona:** Italo Souza (Me)
- **Pets:** 3 dogs

## 4. Scope (The "Contract with Myself")

### In Scope (Must Haves)

- [ ] Feature A: Create pets
- [ ] Feature B: Create pet schedules routines (feeding, medication, walks, etc)
- [ ] Feature C: Create health tracking (weight, vet visits, vaccinations, grooming, etc)
- [ ] Feature D: Create reminders (notifications via sms, email, push)

### Out of Scope (Strictly Forbidden for v1)

- [ ] Complex RBAC - defer to v2
- [ ] Multi-tenant
- [ ] Multi-workspace

## 5. Use Cases & User Journey

### Create a new pet

1. User logs in.
2. User navigates to the "Pet List" page.
3. User clicks on the "Create Pet" button.
4. System displays the "Create Pet" form.
5. User fills the form with the pet's information.
6. System saves the pet's information.
7. System displays the "Pet List" page with the new pet.

### Create a new routine schedule

1. User logs in.
2. User navigates to the "Pet List" page.
3. User clicks on a pet to open its details.
4. User clicks on the "Routine Schedules" tab.
5. System displays the "Routine Schedules" page.
6. User clicks on the "Create Routine Schedule" button.
7. System displays the "Create Routine Schedule" form.
8. User fills the form with the routine's information.
9. System saves the routine's information.
10. System displays the "Routine Schedules" page with the new routine.

### Create a new health tracking

### Create a new reminder
