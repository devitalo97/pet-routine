## 2. Architecture & Design Patterns

- **Pattern:** Clean Architecture within a Monorepo
- **Runtime:** **Bun**
- **Explanation:**
  The project will use a monorepo structure managed by Bun workspaces. The frontend will be a React Application powered by **Vite**, utilizing **TanStack Router** for routing and **server functions** to seamlessly bridge the client and server. **TanStack Query** will handle asynchronous state and data fetching, and **TanStack Table** will be used for complex data grids (e.g., pet lists, routine schedules). Styling will rely on **Tailwind CSS** and **shadcn/ui**.

  The backend logic will reside in a dedicated package within the monorepo, strictly adhering to Clean Architecture principles to isolate domain rules from frameworks. This package acts as a library consumed by the frontend:
  - **`/core/domain`:** Enterprise business rules and pure entities (e.g., `Pet`, `Routine`).
  - **`/core/application`:** Application use cases (e.g., `CreatePetUseCase`, `ScheduleRoutineUseCase`).
  - **`/infrastructure`:** Frameworks and external drivers. This layer will house the **Drizzle ORM** schemas, repositories, and **Supabase** client configurations.
  - **`/adapter`:** Concrete implementations of interfaces defined in the application layer. (e.g., repositories, storages, gateways).

## 3. System Context & Boundaries

- **Frontend App:** React App (Vite) running on **Bun**. It hosts both the client UI and the TanStack Router server functions.
- **Backend Package:** A library of framework-agnostic use cases and infrastructure logic.
- **Database:** **Supabase** (PostgreSQL) managed via **Drizzle ORM**.

## 4. Data Model / Schema (Drizzle Draft)

```typescript
// pets
{
  id: uuid,
  name: varchar,
  species: varchar,
  breed: varchar, // optional
  birthDate: date, // optional
  createdAt: timestamp
}

// routines
{
  id: uuid,
  petId: uuid (FK),
  title: varchar,
  type: enum('feeding', 'medication', 'walk', 'other'),
  schedule: varchar, // cron expression or specific time
  createdAt: timestamp
}

// health_tracking
{
  id: uuid,
  petId: uuid (FK),
  type: enum('weight', 'vet_visit', 'vaccination', 'grooming'),
  notes: text,
  logDate: date,
  createdAt: timestamp
}

// reminders
{
  id: uuid,
  petId: uuid (FK),
  routineId: uuid (FK, optional),
  message: varchar,
  remindAt: timestamp,
  status: enum('pending', 'sent', 'cancelled'),
  createdAt: timestamp
}
```
